First, bit of history:

I first did the MPICH2 import into git from the tarballs.  I used the .cvsignore files to delete all the generated files and generate .gitignore files for our own use.  After the switch to SVN, there were no .cvsignore files, but I was able to use the puplic SVN.  I changed the script to generate the .gitignore files by querying the svn:ignore property, and left the part that deletes ignored files--I was assuming that they were generated.

I was having some intermittent trouble running "maint/updatefiles", because it was complaining about a missing ui/mpiexec/Makefile.mk file in src/pm/hydra.  I finally realized that it's entire directory is ignored:

$ svn propget svn:ignore

mpiexec
*.gcov
*.gcda

So, I hacked together a shell script to enumerate ignored files that are part of the SVN tree and also ignored.  It can be run in any part of a freshly checked-out tree.

for dir in $(find -type d | grep -v '\.svn')
do
    (
        cd $dir &&
        for ignore in $(svn propget svn:ignore)
        do
            if [[ -e $ignore ]]
            then
                readlink -f $ignore
            fi
        done
    )
done

It lists:

trunk/test/mpi/f90/timer/testlist

Is that something that should be changed?  It doesn't seem like a good idea to ignore files that are checked in, but maybe SVN handles that.  I don't know about git.
