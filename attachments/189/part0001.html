  Yes we have changed the way singleton init client works with SMPD. Now when a singleton init client is run it tries to connect to the process manager (without this change the process has no support from the PM and calls like MPI_Comm_spawn() won't work).
  Is there any reason why you don't want smpd to be running on these machines ?
