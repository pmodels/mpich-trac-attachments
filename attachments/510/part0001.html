Dunno if you all were going to reply to my previous mail, but here is status:

Ok, I managed to hand-install mpich2 (which led to the same error). The real issue is that python2.5 is not compatible because of the md5 issue. But al long as 2.5 is available it (mpich2 install) finds it. The install guide gives --with-python  bas a option to configure, but this does not exist anymore. So I had to disable 2.5 by hand, then it finds python2.4 and then it all works. 

So regardless of macports, mac etc you need to:

1. check about the md5 issue with python 2.5. I really don't know why it work on your systems with python 2.5, when md5 was deprecated by that version. 
2. Either change the install guide - maybe it already has been, I just have an older version, or add back --with-python option.
3. (Not your issue) I believe if 1 was fixed, then the macports install would work fine too.

Thanks,
Anil
PS FYI my config is Intel-Mac, Core 2 Duo, 10.5.6. Python 2.4, gfortran, gcc4.2
