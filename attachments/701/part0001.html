 Take a look at the section 9.8 in the window's developer's guide (http://www.mcs.anl.gov/research/projects/mpich2/documentation/index.php?s=docs) on debugging MPI programs.

# You can also attach to MPI programs that are running using the VS debugger ("Attach to process").
# If all the MPI processes are launched on the same machine/node then you can start smpd in debug mode (smpd -stop; smpd -d;) and use multiple instances of VS to start your MPI program (mpiexec -n 2 devenv.exe fpi.exe ==> will start two VS windows and you can step individually through them).

(PS: The debug versions of MPICH2 libraries, eg: fmpich2d.lib, is only required if you are debugging the MPICH2 library.)
