I'm forwarded this to the MPICH2 bug list.  Normally, the best way to run two different executables is to use the mpiexec syntax for that which uses a colon ( : ) to separate the executables.  As long as the byte order and datatype sizes are the same, it should be possible to run these executables with MPICH2.

Bill
