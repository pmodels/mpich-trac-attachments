I tried submitting the following on your Wiki, but it was rejected as potential spam.

I have found out the causes of the problem.  In short, the causes are an unclear procedure / documentation, necessary software not obviously available, and bugs in the installers that will accept a bad initial state.

I found out by trial and error that there are two install files that must be processed in order.
First, one must install
mpich2-1.0-1-win32-ia32.msi, which is not obviously available on your download page.
That puts some essential stuff in place which is necessary to then install
mpich2-1.1.1p1-win-ia32.msi

Yes, there were too copies if MPICH2 on the Add/ Remove programs page, indistinguishable except by clicking "support information".  See below for a suggestion on how to fix this.

The problem was caused by the fact that when I deinstalled the older version (1.0), I removed the base install with its essential stuff.  Then I could not deinstall or reinstall mpich2-1.1.1p1-win-ia32.msi, which (as I found later) is a patch and not a complete installation.

So, finally I reinstalled the two installs in the order above, and it worked.

Below is what I would suggest to fix this problem:

Have a link to both the base install and the patch in the same place on your web download site.

Provide clear IMPORTANT instructions on the same web page stating that the base install must be installed first and the patch installed second.

Instead of simply "MPICH2" labels for both base install and patch in the Add / Remove programs window, provide descriptive labels, such as:

MPICH2 base 1.0, install first, uninstall last

MPICH2 patch 1.1.1p, install last, uninstall first

Don't let the patch installer work unless it finds a compatible base install.  If it doesn't find one, provide a descriptive message of the problem and how to fix it, not the obscure message I got (see original)

Don't allow uninstallation of the base install if there is a patch still installed.  Provide a descriptive message as above.

Another option, which makes more sense to me, would be to release in complete versions and abandon the patch concept altogether.

  Try creating a restore point to backup your registry before manually
 editing it ( http://support.microsoft.com/kb/322756).
  The places in registry that you might want to check for MPICH2's presence
 (after backing up your registry),

 {{{

 HKEY_LOCAL_MACHINE\SOFTWARE\MPICH

 HKEY_LOCAL_MACHINE\SOFTWARE\MPICH2

 HKEY_CURRENT_USER\SOFTWARE\MPICH

 HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\ =>
 This directory will contain a directory (a unique GUID) containing MPICH2
 uninstall info.

 HKEY_LOCAL_MACHINE\System\CurrentControlSet\Service\mpich2_smpd

 }}}
